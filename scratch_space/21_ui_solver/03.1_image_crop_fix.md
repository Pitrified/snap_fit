## Root Cause Analysis

`get_piece_img()` in `piece_service.py` loads the **original** sheet photo
(before ArUco rectification and cropping) but uses `sheet_origin` and
`contour_region` coordinates that live in the **cropped-sheet** coordinate space
(after rectification + symmetric crop + QR strip crop). These coordinate spaces
are incompatible; the original photo has different resolution and perspective.

Additionally, the crop-dimension formula `padded_w = 2 * cx + cw` only works
when `pad_rect()` applies symmetric padding without edge clamping. For pieces
near the sheet boundary, the formula gives wrong dimensions.

## Fix Plan

### Phase 1: Documentation (done)

1. Created `docs/guides/coordinate_spaces.md` explaining all 4 coordinate
   spaces, the transformation chain, and rules.
2. Updated `.github/copilot-instructions.md` with a section referencing the doc.
3. Wrote this plan.

### Phase 2: Data model - add `padded_size` to PieceRecord

Store the actual piece image dimensions `(width, height)` so the crop can be
reconstructed exactly without approximation formulas.

- Add `padded_size: tuple[int, int] = (0, 0)` to `PieceRecord`.
- In `PieceRecord.from_piece()`, set it from `piece.img_orig.shape[:2]`
  (reversed to width, height).
- Add SQLite migration `ALTER TABLE pieces ADD COLUMN padded_size TEXT`.
- Update `_piece_to_row` / `_row_to_piece` in `sqlite_store.py`.

### Phase 3: Save processed sheet images at ingest time

The rectified+cropped sheet image only exists in memory during ingest. Save
it to disk so `get_piece_img()` can load it later.

- Add `SheetManager.save_sheet_images(sheets_dir)` that writes each sheet's
  `img_orig` as JPEG to `sheets_dir/{sheet_id}.jpg`.
- Call it in `PieceService.ingest_sheets()` after building pieces.

### Phase 4: Fix `get_piece_img()` crop logic

- Load the processed sheet image from `cache/{tag}/sheets/{sheet_id}.jpg`
  instead of the original photo.
- Crop using: `sheet_img[y0 : y0 + ph, x0 : x0 + pw]` where
  `(x0, y0) = sheet_origin` and `(pw, ph) = padded_size`.
- Fall back to the `contour_region` formula if `padded_size` is `(0, 0)`
  (old data).

### Phase 5: Tests

- Unit test `PieceRecord` with `padded_size`.
- Test `save_sheet_images` / round-trip.
- Test `get_piece_img()` returns correct PNG bytes from processed sheet.
- Run full verification suite.
