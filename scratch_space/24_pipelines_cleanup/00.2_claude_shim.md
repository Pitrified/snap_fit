# Phase 0 - CLAUDE.md shim (prerequisite)

## Overview

A one-file prerequisite done before any pipeline work, so the repo-level
conventions this effort adds (D20) actually reach Claude Code. The repo has
`AGENTS.md` and `.github/copilot-instructions.md` but no `CLAUDE.md`, so Claude
Code auto-loads none of them today.
Context: [00_start.md](00_start.md), decision D21 (from Q14).

## Goals

1. Make the repo's existing instruction files reach Claude Code.
2. Do it as an isolated, independently-mergeable change.

## Plan

1. Add `CLAUDE.md` at repo root whose body imports the shared instructions:

   ```markdown
   @.github/copilot-instructions.md
   ```

   This matches the user's global convention (a repo `CLAUDE.md` that imports
   the canonical `.github/copilot-instructions.md`), and surfaces every existing
   repo instruction to Claude Code, not only the notebook convention.
2. Confirm the import resolves (the referenced file exists and is non-empty).
3. Commit on its own on the current `feat/pipeline-sketch` branch (D21). That
   branch is short-lived and merges to main soon, so the shim reaches main
   quickly without a separate branch.

## Out of scope

- Writing the agent-notebook convention text itself. That is added to
  `.github/copilot-instructions.md` (and `AGENTS.md`) in phase 1; this phase
  only wires the file up so that text will be seen.
- Any `pipelines/` content.

## Done when

- `CLAUDE.md` exists at repo root and imports `@.github/copilot-instructions.md`.
- It is committed as its own commit on `feat/pipeline-sketch`.
