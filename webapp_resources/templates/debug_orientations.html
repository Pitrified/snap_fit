Orientation Debug

Each piece shown in all 4 orientations. Compare backend-rotated (via ?orientation= query param) against CSS rotation to verify they match. Edge shapes shown per rotated position help confirm the get_original_edge_pos mapping is correct.

Convention reference

  • DEG_0 - Original photo orientation. TOP edge is physically at top.
  • DEG_90 - Rotated 90° CW. Original TOP now faces RIGHT.
  • DEG_180 - Rotated 180°. Original TOP now faces BOTTOM.
  • DEG_270 - Rotated 270° CW (= 90° CCW). Original TOP now faces LEFT.

Verification: pick a CORNER or EDGE piece with a visible flat side. In DEG_0 the flat side should be on the expected edge. In DEG_90 the same flat side should be rotated 90° clockwise. Backend-rotated and CSS-rotated images must be visually identical.
